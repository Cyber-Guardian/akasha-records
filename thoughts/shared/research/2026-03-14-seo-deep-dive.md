# FileScience.io SEO Deep Dive

## Executive Summary

FileScience.io is essentially **invisible organically**. The domain ranks #6,720,264 in the US with ~18 organic visits/month, all from branded searches ("filescience"). ## 4. Strategic Recommendations

### Tier 1 — Quick Wins (0-30 days)

**A. Optimize existing cloud pages for target keywords**
The `/clouds/clio/`, `/clouds/microsoft-365/`, `/clouds/box/` pages exist but rank for nothing. Each needs:
- Keyword-targeted meta title (e.g., "Clio Backup & Data Protection | FileScience")
- Keyword-rich H1 and subheadings
- 1,000+ words of substantive content covering features, use cases, compliance
- FAQ section targeting question keywords
- Schema markup (Product, FAQ)

**B. Create a dedicated "Clio Backup" page/pillar**
KD 0. FileScience IS the Clio backup product. This should be a #1 ranking with minimal effort. Expand beyond the current cloud page into a comprehensive resource.

**C. Get listed on SaaS review platforms**
G2, Capterra, TrustRadius, GetApp — each provides a high-authority backlink + direct referral traffic + social proof.

**D. Clio App Directory / Partner Page**
If not already listed, this is a high-quality backlink + qualified referral traffic from Clio's user base.

## 5. Technical SEO Notes

- **PDF ranking:** `onboarding.pdf` ranks for 2 keywords and drives 11% of traffic. Create proper HTML landing pages to capture this intent.
- **Duplicate pages:** `/clouds/microsoft-365/` appears twice in organic results (positions 17 and 20) — potential canonicalization issue.
- **Dashboard subdomains:** `dashboard.filescience.io` and `demo.dashboard.filescience.io` are indexed — ensure these have proper noindex/robots if they shouldn't be in SERPs.
- **Structured data:** Add Organization, Product, FAQ, and BreadcrumbList schema.
- **Industries pages:** `/industries/`, `/industries/healthcare-medical/`, `/industries/architecture-engineering/` — these need content or should be consolidated. Thin pages hurt crawl budget.
