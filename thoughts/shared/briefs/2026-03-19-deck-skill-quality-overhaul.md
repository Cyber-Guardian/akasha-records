# Idea Brief: Deck Skill Quality Overhaul

**Date:** 2026-03-19
**Status:** Shaped → Planning

## Problem

Decks generated by `/deck` are functional but visually flat — every slide looks identical (white background, left-aligned text, same font sizes). There's no visual rhythm, no emphasis hierarchy, and no branded identity. Content regularly overflows slides because Marp has no vertical auto-shrink (confirmed: closed as "not planned" by maintainers). The result reads like a formatted document, not a presentation suitable for C-suite/leadership stakeholders.

## Constraints

- Marp v4 is the renderer — CSS + markdown directives only (no JavaScript, no animations)
- PDF output via Chromium — SVG foreignObject is the hard clip boundary, not CSS overflow
- No vertical auto-shrink exists in Marp — content discipline in SKILL.md is the primary overflow defense
- Custom theme must include `@auto-scaling fittingHeader` metadata or `<!-- fit -->` headings silently do nothing
- `<div>` tags work without `--html` flag in v4 (in default allowlist)
- Blank lines required between `<div>` and markdown content (CommonMark spec)
- Dual PDF+HTML output requires two CLI invocations (not one)
- Brand colors: primary purple `#6d59c7` / `#4b3f8c`, secondary rose `#c15074`, tertiary blue `#4a8dd0`, text `#1c1c1c`, light purple `#f0f2ff`
- Font: Inter (already in use)

## Options Considered

### AI-Generated Slide Illustrations
Add an image generation step to render_deck.py (DALL-E 3 / Replicate) that generates contextual visuals from Claude-authored prompts embedded in the markdown.
- Gains: Every deck gets custom visuals, impressive output
- Costs: API cost per render, 5-15s added latency per image, quality variance, pipeline complexity
- Complexity: High
- **Decision: Deferred** — tracked in Linear backlog

### Curated Brand Asset Library
Commission/source 15-20 SVG illustrations, Claude picks by tag.
- Gains: Consistent quality, zero runtime cost
- Costs: Upfront creation effort, limited coverage
- Complexity: Medium
- **Decision: Deferred** — can layer on later without pipeline changes

### CSS + SKILL.md Overhaul (chosen)
New branded theme CSS with 6 slide classes, content-driven layout instructions, narrative arc templates, Mermaid brand theming, and subtle background textures.
- Gains: Immediate visual improvement, zero new dependencies, brand-coherent
- Costs: Two files to rewrite (CSS + SKILL.md), a few SVG background assets to create
- Complexity: Low-Medium

## Chosen Approach

**CSS + SKILL.md Overhaul** — highest leverage for lowest complexity. Addresses both the visual flatness (branded slide vocabulary, background textures, Mermaid theming) and the content overflow problem (layout-by-content-type, narrative arc templates). No new runtime dependencies.

### What's included:

**A. Branded Slide Vocabulary (CSS)**
- 6 slide classes: `lead`, `section`, `highlight`, `diagram`, `dense`, `split`
- FileScience brand colors throughout
- `@auto-scaling fittingHeader` metadata
- "N / Total" pagination via `section::after`
- `.lead` with purple gradient background

**B. Content Architecture (SKILL.md)**
- Map content types to layouts (comparison → two-column, key metric → highlight, diagram → own slide)
- Hard overflow rules: 4 bullets max, diagrams get own slides, tables max 6 rows
- Two-column layout instructions using `<div class="cols">` with blank line convention

**C. Narrative Arc Templates (SKILL.md)**
- Pitch arc: Hook → Insight → Market → What it does → Why it works → Ask
- Progress arc: Where we were → What changed → What we learned → What's next
- Decision arc: Situation → Options → Recommendation → What we need

**D. Typography Emphasis (CSS)**
- `.stat` class for hero numbers (large font, rose accent)
- `.quote` class for pull quotes (left border, italic)
- Lighter gray for supporting text, darker for key points

**E. Enhanced Mermaid Theming**
- Pass FileScience brand colors to mermaid.ink via theme config parameter
- Purple/rose nodes instead of default gray

**F. Branded Background Textures**
- 3-4 subtle SVG background patterns in brand colors
- Used on section dividers and highlight slides via `![bg]`

## Key Context Discovered During Shaping

- Marp v4 changed `section` to `display: block` — improves CSS Grid compatibility, vertical centering uses `align-content: safe center` (overflow goes to bottom only)
- `<!-- fit -->` requires `@auto-scaling fittingHeader` in theme CSS metadata comment — our custom theme lacks this (scripts/deck_theme.css:1)
- `![bg right:40%]` split backgrounds confirmed working with `--allow-local-files` (already in our build_marp_command at scripts/render_deck.py:109)
- CSS Grid two-column layouts work via `<div class="cols">` — blank line after opening `<div>` is mandatory per CommonMark
- Mermaid.ink supports theme parameter for custom styling
- `section::after` must include `attr(data-marpit-pagination)` or Marpit ignores the content declaration entirely
