## Learnings

- hatch-polylith-bricks only copies brick source into wheel — does NOT install third-party deps. Triage agent needs two-step build (wheel + platform deps install).
- aws-durable-execution-sdk-python must NOT be in dev deps — it activates @durable_execution decorator which expects Lambda-specific event format, breaking tests. Tests rely on ImportError fallback making it a passthrough.
- boto3 constraint in triage-agent pyproject.toml must be relaxed to >=1.40.0 (not >=1.42.1) to avoid conflict with aioboto3 in the root workspace.
- git mv with shell globs doesn't work reliably — use explicit file paths or loops over git ls-files output.
- When working across branches, be very careful with stash/checkout/cherry-pick — easy to lose work or commit to wrong branch. Prefer staying on one branch.
- Edit tool changes can be reverted by linter hooks that re-read files. For bulk mechanical changes (like sed across many files), use Python scripts instead of sed to avoid partial matches.
